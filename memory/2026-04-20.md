## 自动化尝试 - Chrome/Playwright

### 尝试结果
- **Chrome**: AppleScript JS返回0，无法获取树节点
- **Playwright/Chromium**: 安装成功(playwright-1.58.0)，但懒加载问题相同

### 根因分析
ComfyUI侧边栏使用 **PrimeReact p-tree** 组件，是**懒加载(lazy-loading)**的：
- 只有用户真正点击/交互后才会渲染子节点
- 自动化工具(Safari/Chrome/Chromium)的模拟点击无法触发
- Safari例外：之前手动展开过，树状态被缓存，但刷新后会重置

### DOM结构发现
```
DIV.comfy-vue-side (侧边栏容器)
  └─ DIV.p-scrollpanel
       └─ DIV.p-scrollpanel-content-container
            └─ DIV.p-tree (主树组件)
                 └─ UL.p-tree-container
                      └─ LI.p-tree-node (叶节点，包含.json文件)
```

### 最终结论
| 方案 | 状态 | 说明 |
|---|---|---|
| Safari手动展开 | ⚠️ 临时 | 刷新后需重新展开 |
| Chrome自动化 | ❌ | 不支持/do JavaScript |
| Playwright | ❌ | 懒加载无法绕过 |
| API直接调用 | ✅ | 完全正常 |

### 当前状态
- **工作流文件**: 48个（已保存在 ~/ComfyUI-Workflows/）
- **API状态**: 正常（刚测试生成420KB图片成功）
- **自动化**: 受限，需手动触发懒加载

### 经验教训
1. **浏览器自动化不等于真实用户** — 很多网站用懒加载防爬
2. **Safari的localStorage缓存是唯一例外** — 之前手动展开过的状态会被缓存
3. **Playwright截图功能很实用** — `page.screenshot()` 可以查看页面实际渲染状态
